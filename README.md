# sql_pro_university
This project involves the design and implementation of a relational database system for managing university operations using MySQL. The database is structured with multiple interconnected tables, including courses, departments, enrollments, professor, and student_table, to efficiently store and manage academic and administrative data.

The primary objective of this project is to create a well-organized and normalized database that ensures data integrity, reduces redundancy, and supports efficient querying. Each table serves a specific purpose: the departments table stores information about various academic departments; the courses table contains course details linked to respective departments; the professor table maintains faculty records; and the student_table holds student-related information such as personal details and academic data. The enrollments table acts as a bridge between students and courses, capturing which students are enrolled in which courses, thereby implementing many-to-many relationships.

To enhance data consistency and reliability, primary keys and foreign keys are used to establish relationships between tables. Various SQL operations such as SELECT, JOIN, GROUP BY, and aggregate functions are applied to retrieve meaningful insights, such as student enrollment trends, course popularity, and department-wise performance. The project also includes data manipulation techniques like INSERT, UPDATE, and DELETE to manage records effectively.
